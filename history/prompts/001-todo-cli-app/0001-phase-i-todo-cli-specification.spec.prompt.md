## Prompt

/sp.specify Phase I: Todo In-Memory Python Console App CLI

Target audience: Hackathon participants building the first phase of "The Evolution of Todo" project; AI-generated code users and reviewers

Focus: Build a command-line todo application that stores tasks in memory and implements all basic-level features using spec-driven development with Claude Code and Spec-Kit Plus

Success criteria:
- Implements all 5 core features: Add Task, Delete Task, Update Task, View Task List, Mark Task as Complete/Incomplete
- Correctly maintains task data in memory (no persistence)
- Provides clear and user-friendly CLI output for all commands
- Follows clean code principles and proper Python project structure
- All generated code adheres strictly to Phase I Constitution

Constraints:
- Technology stack: Python 3.13+, UV, Claude Code, Spec-Kit Plus
- Data storage: strictly in-memory; no files, databases, or external storage
- CLI-only interface; no GUI or web interfaces
- Generated code must be deterministic and explicit
- All tasks must have unique identifiers, title, description, and completion status

Not building:
- Phase II+ features (web interface, database persistence, AI agents, cloud deployment)
- Recurring tasks, reminders, notifications, or advanced task analytics
- Any manual coding; all implementation must be generated by Claude Code
- Networking, API endpoints, or external integrations

Deliverables:
- Specification file stored in `/specs/phase-1/` folder
- GitHub repository with `/src` folder containing generated Python source code
- README.md with setup and CLI usage instructions
- CLAUDE.md with instructions for Claude Code generation

Timeline: Complete Phase I development and code generation within the hackathon schedule
